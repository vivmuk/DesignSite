Getting Started

Introduction

Nexus CLI is the command-line interface for deploying and managing applications on the Nexus platform. It provides a fast, scriptable surface for every workflow—from development to production.

Installation

Install Nexus CLI globally using your preferred package manager. The CLI requires Node.js 18+ or Bun.

Terminal copy 
npm install -g @nexus/cli

Alternatively, use bun or pnpm:

Terminal copy 
bun add -g @nexus/cli
# or
pnpm add -g @nexus/cli

ℹ If you see a permission error on macOS, prefix with sudo or install to a user-owned directory. We recommend using fnm or nvm to manage Node versions.

Your first deployment

1

Authenticate

Link your Nexus account. This opens a browser window for OAuth and saves a token to ~/.nexus/credentials.

Terminal
nexus login
2

Initialize your project

Run nexus init from your project root to generate a nexus.config.ts and link to a deployment target.

Terminal
nexus init --framework next
3

Deploy

Ship to production. The CLI builds, uploads, and returns a deployment URL in seconds.

Terminal
nexus deploy --prod

CLI flags

Global flags that apply to all commands.

FlagAliasDescription
--help-hShow help for a command
--version-vPrint CLI version
--token <tok>-tAPI token (overrides ~/.nexus/credentials)
--jsonOutput results as JSON
--no-colorDisable ANSI color output
--debugEnable verbose debug logging

Configuration file

Nexus reads nexus.config.ts at the project root. It is fully typed and supports all platform options.

nexus.config.ts copy 
// nexus.config.ts
import { defineConfig } from '@nexus/config'

export default defineConfig({
  framework: 'next',
  region: 'iad1',    // us-east-1 (default)
  env: {
    NODE_ENV: 'production',
  },
  routes: [
    { src: '/api/(.*)', dest: '/api/$1' },
  ],
})
